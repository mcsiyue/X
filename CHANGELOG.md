# 更新记录

## v3.6.0

- 左上角应用名称改为“月之视频下载器 v当前版本号”。
- 窗口标题和提示框标题同步使用“月之视频下载器”。

## v3.5.0

- 新增“重新下载失败视频”按钮。
- 失败重试次数默认值改为 5。
- 移除“自动选择最高画质”复选框，程序默认始终使用最高画质。
- 右键菜单新增“删除已下载视频文件”。
- 标题更新为 Twitter + PornHub + Truvaze。

## v3.4.0

- 新增 Truvaze 站点支持。
- 支持 `truvaze.com` / `www.truvaze.com` 视频页链接。
- 通过 JSON-LD `VideoObject.contentUrl` 和页面中的 `video.twimg.com` mp4 直链提取视频。
- 站点识别新增 `Truvaze` 标注。

## v3.3.0

- 新增站点自动识别功能。
- 添加任务时根据链接域名自动标注 `X/Twitter`、`PornHub` 或其它 hostname。
- 任务列表新增“站点”列，方便混合队列中区分视频来源。

## v3.2.0

- 优化停止下载响应：点击停止后立即取消排队任务，m3u8 分片等待改为短间隔轮询。
- 缩短直链和 m3u8 分片网络超时，减少卡住很久才停止的问题。
- 新增“下载速度”列，显示当前下载速度。
- “下载进度/大小”列显示 `已下载大小/总大小`。
- 文件大小扫描到以后立即显示到“文件大小”列。

## v3.1.0

- 优化 PornHub m3u8 下载进度显示：明确显示“解析 m3u8 列表”“下载分片 x/y”“合并分片 x/y”“写入最终文件”。
- m3u8 下载分片阶段进度推进到 90%，合并阶段推进到 99%，任务真正完成后才显示 100%。
- 批量下载运行中继续添加的新任务会自动加入当前下载队列。
- 下载队列空闲后会短暂等待，避免刚添加的新任务漏处理。
- 加宽状态列，方便查看当前操作。

## v3.0.0

- 新增 PornHub 视频解析和下载支持。
- 支持 `pornhub.com`、`www.pornhub.com`、`cn.pornhub.com`、`m.pornhub.com`。
- PornHub 解析逻辑从页面 `flashvars` / `mediaDefinitions` 提取 mp4 或 m3u8 视频地址。
- PornHub 默认选择最高画质。
- 同清晰度下优先选择 mp4/直链，减少不必要的 m3u8 分片下载。
- m3u8 下载从串行分片改为并发分片，默认 8 个分片线程。
- 直接下载读取块提升到 1MB，减少循环和 IO 开销。
- 更新 GitHub 发布源码文档，新增 `DOWNLOAD_LOGIC.md`。
- 继续支持 PyInstaller 单文件 exe，普通用户无需安装 Python。

## v2.8.0

- 进一步修复 Windows 任务栏图标显示为 Tk 羽毛的问题：设置外层/root HWND、窗口类图标和 AppUserModelID。

## v2.7.0

- 自动提取改为逐任务入队：每新增一个任务就立即加入后台自动提取队列，不再等待整批添加完成。
- 自动提取队列会按“并发线程”设置启动多个后台提取线程。
- 修复 Windows 任务栏运行图标可能显示 Tk 默认羽毛图标的问题，强制使用 `推特下载器图标.ico`。

## v2.6.0

- 新增“添加队列后自动提取”：添加链接后自动解析标题、画质、直链和文件大小。
- 新增下载总文件大小统计。
- 下载失败任务自动标红。
- 新增“重试失败”按钮和右键菜单项。
- 新增表头排序，支持按下载进度、文件大小等字段排序。

## v2.4.0

- 新增多线程并发下载，界面可设置并发线程数，默认 3 个线程。
- 新增“文件大小”列，下载时自动读取并显示视频大小。
- 新添加的任务现在会显示在任务列表最上方。
- 优化并发下载时的文件命名，避免同名文件冲突。

## v2.3.1

- 修复关闭到托盘后点击托盘图标无法恢复窗口的问题。
- 新增托盘右键菜单：显示窗口、退出。
- 添加自定义程序图标、窗口图标和托盘图标。

## v2.2

- 批量视频下载基础功能。
- 任务列表、进度展示、错误日志、保存目录选择等功能。
